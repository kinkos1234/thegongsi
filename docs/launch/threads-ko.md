# Threads 한국어 포스팅 초안

## 포스팅 1 (메인 훅 — 비개발자도 OK)

```
네이버 증권 매일 봐도 삼성전자 공시 중 뭐가 "진짜 이상한지" 모르겠다.

DART에 연 100만 건 공시가 올라오는데 99%는 루틴(정기보고서, 배당결정 같은 거)이고 1%가 상장폐지·감사거절·최대주주변경처럼 투자자를 직격한다.

그 1%만 골라내는 도구가 없어서 2주 동안 만들었다.

이름은 The Gongsi · 더공시.
```

## 선택 포스팅 4 (개발자 팔로워 대상, 기술 훅)

```
처음엔 교과서처럼 pgvector + SBERT로 의미 검색 썼다.

근데 한국 금융 도메인(약칭, 계약명, 티커)에 임베딩이 약해서 "HBM 공급망" 같은 개념적 질의가 안 됐다.

Claude tool_use로 3개 툴(Cypher + 키워드 + 회사 검색) 던져주고 agentic loop 돌리니 벤치 recall 0.75 → 1.0, precision 0.71 → 1.0.

한국어 금융 RAG는 임베딩보다 agent가 낫다는 결론.
```

## 포스팅 전략

**순서:**
1. 포스팅 1 먼저 (훅 만 올리고 반응 봄)
2. 30분~1시간 후 포스팅 2를 **댓글**이 아닌 **연결 답글**로 달기 (Threads에서 thread 걸기)
3. 포스팅 3을 thread 이어가기
4. 개발자 반응 있으면 포스팅 4 추가

**팁:**
- 첫 포스팅에 **링크 없이** 훅만 → 알고리즘이 좋아함 (외부 유도 없는 초기 engagement)
- 이미지·스크린샷 붙이기 (홈페이지 또는 삼성전자 대시보드 캡처) — engagement 2~3배 up
- 좋아요 먼저 눌러주는 팔로워에게 답글로 감사 → 알고리즘 부스트
- 1시간 안에 다시 보고 댓글 단 사람에게 답장해주기

**주의:**
- "제가 만들었어요 도와주세요" 톤 금지 → "알아두면 좋은 거 발견했어요" 톤
- 링크는 3번 포스팅까지 미루기 (첫 2개는 순수 콘텐츠)
- 해시태그 2~3개만 (`#한국주식` `#DART공시` `#오픈소스` 정도)
